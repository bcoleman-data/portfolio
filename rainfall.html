Rainfall Prediction

This project explores the challenge of predicting rainfall using historical NOAA weather data. As part of a collaborative bootcamp team, I helped design a full pipeline—from data acquisition and cleaning to machine learning and a user-facing web interface.

Project Overview

We began by leveraging Google BigQuery to merge and clean weather data from multiple U.S. states. After exploring the dataset, we narrowed our focus to Missouri to build more targeted training and testing datasets.

Using SQLite for backend storage and Scikit-Learn for modeling, we implemented a multivariate linear regression model to predict rainfall. The model achieved approximately 70% accuracy.

To make the model accessible, we developed a small web application using HTML, CSS, JavaScript, and Flask, allowing users to interact with the model and generate predictions.

Data Pipeline

  • Queried NOAA GSOD weather data using Google BigQuery
  • Merged 29 years of multi-state datasets (1990–2019)
  • Reduced dataset from 21M rows to 8M rows across 11 states
  • Filtered to Missouri for machine learning
  • Stored cleaned data in SQLite

Machine Learning

  • Logistic Regression to predict rainfall levels
  • Latitude and longitude used to identify location
  • Month and day used to capture seasonal patterns
  • Achieved approximately 70% accuracy
  • Model saved using the joblib module

Frontend & Deployment

  • Connected Flask to SQLite backend
  • Loaded machine learning model server-side
  • Created endpoints to render web pages
  • Used D3.js to update a dynamic weather card
  • Embedded visualizations and model outputs into HTML/CSS pages
