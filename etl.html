ETL Pipeline Project

Overview

This project demonstrates a complete Extract‑Transform‑Load (ETL) workflow built to clean, normalize, and load raw data into a structured relational database. The pipeline automates data ingestion, performs multi‑step transformations, and outputs a clean dataset ready for analysis and reporting.

Data Extraction

The pipeline begins by importing raw Denver Crime CSV files from Kaggle. These files contain inconsistencies, missing values, and mixed data types that require systematic cleaning before loading.

❮ ❯

Data Transformation

The transformation stage includes:

  • Handling missing or null values
  • Standardizing column names and data types
  • Normalizing categorical fields
  • Joining related datasets
  • Removing duplicates and invalid records

❮ ❯

Loading Into the Database

After transformation, the cleaned dataset is loaded into a PostgreSQL database. The schema is designed to support efficient querying and downstream analytics.

Schema Design

The final database schema organizes the cleaned data into normalized tables with clear relationships. This structure supports analytical queries and dashboarding.

Analysis & Outputs

Once loaded, the data can be used for reporting, dashboards, or further modeling. This project demonstrates how a well‑designed ETL pipeline improves data quality and enables reliable analytics.

Technologies Used

  • Python (Pandas)
  • PostgreSQL
  • SQL
  • Jupyter Notebook
