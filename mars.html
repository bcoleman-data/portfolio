Mission to Mars

A web application that scrapes live Mars data from multiple NASA and astronomy sources, stores it in MongoDB, and displays it in a dynamic Flask-powered dashboard.

Overview

This project automates the process of gathering Mars-related data from several external websites and compiles it into a single, interactive dashboard. The workflow includes web scraping, data storage, and dynamic rendering through Flask.

Data Sources Scraped

Using BeautifulSoup, Splinter, and Requests, I scraped:

  • Latest NASA Mars news headline and summary
  • Featured Mars image from JPL
  • Recent Mars weather report
  • Mars facts table (via Pandas)
  • High-resolution hemisphere images from USGS

Technical Workflow

1. Web Scraping

Automated scraping routines were developed in a Jupyter Notebook and later modularized into a Python script (scrape_mars.py) that returns a structured dictionary of all collected data.

2. Data Storage

Scraped data is stored in MongoDB, with each new scrape replacing the existing document to keep the dashboard up to date.

3. Flask Application

A Flask app serves the dashboard, pulling the latest data from MongoDB and rendering it through a Jinja2 template.

Dashboard Features

  • Latest Mars news article
  • Featured Mars image
  • Mars weather report
  • Mars facts table
