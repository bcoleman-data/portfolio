Employee Database Project

Overview

This project involved designing a complete relational employee database from six CSV files containing information on employees, departments, salaries, titles, and organizational structure. I built the schema in SQL, created an Entity Relationship Diagram (ERD), and performed analytical queries to uncover insights about hiring trends, departmental composition, and salary distributions.

Data Sources

The project was built from six raw CSV files:

  • employees.csv
  • departments.csv
  • dept_emp.csv
  • dept_manager.csv
  • salaries.csv
  • titles.csv

Database Schema

I designed a normalized schema that accurately models the relationships between employees, departments, managers, salaries, and titles. The ERD below illustrates the final structure.

Key SQL Queries

To analyze the dataset, I wrote SQL queries that explored:

  • Employee salary information joined across multiple tables
  • Hiring patterns, including employees hired in specific years
  • Department manager assignments and their employment timelines
  • How employees are distributed across departments like Sales and Development
  • Last‑name frequency to identify common patterns in the workforce

Insights

Through this analysis, I uncovered patterns in salary progression, departmental growth, and employee movement across roles. These insights demonstrate how relational databases can support HR analytics and organizational decision‑making.

Technologies Used

  • PostgreSQL
  • SQL
  • QuickDBD (for ERD)
  • Python (optional for data prep)
